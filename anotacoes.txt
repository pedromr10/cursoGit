--- COMANDOS ---
* Para iniciar um rep: git init
* Para adicionar arquivos: git add nomeDoArquivo
* Para adicionar arquivos dentro de pastas: git add .\css\styles.css
* Para adicionar todos os arquivos que estavam esperando para serem adicionados: git add .

* Para commitar com um comentário: git commit -m "primeiro commit"
* Para commitar um arquivo em específico: git commit a.txt -m "Enviando o arquivo a"
* Para commitar vários arquivos de uma vez: git commit -a -m "enviando varios arquivos de uma vez"

* Para renomear a branch atual: git branch -M main
* Para adicionar um rep remoto chamado origin com o endereço dado: git remote add origin https://github.com/pedromr10/cursoGit.git
* Para enviar a branch main para o remoto origin e a define como padrão para futuros pushes/pulls: git push -u origin main
* Para enviar arquivos para o repositório remoto: git push
* Para receber arquivos do rep remoto para o local: git pull

* Para clonar um rep: git clone https://github.com/pedromr10/cursoGit.git
* Para clonar um rep no diretório atual: git clone https://github.com/pedromr10/cursoGit.git .

* Para remover um arquivo: git rm nomeDoArquivo

* Para ver o log / modificações realizadas no rep: git log
* Para sair da visualização do log: q

* Para mudar a localização de um arquivo (git mv nomeDoArq ondeVaiArq): git mv .\rodape.css .\css\rodape.css
* Para mudar o nome de um arquivo (git mv dirAntigo dirNovo): git mv .\css\rodape.css .\css\rodapeAlterado.css

* Para desfazer alterações: git checkout .\teste.txt
 
* Para resetar uma branch: git reset --hard origin/master

* Para visualizar os branchs disponíveis: git branch
* Para criar uma nova branch: git branch nomeDaBranch

--- INFOS ---
* git push envia os arquivos commitados;
* o add é utilizado apenas para novos arquivos, se um arquivo já existir e apenas for modificado, usar o commit;
* o link para clonar o rep está na aba "<> CODE" do repositório remoto no github;
* mesmo depois de deletar um arquivo, para que seja transmitido ao master, deve-se dar commit e push;
* depois de renomear ou mover um arquivo, deve-se commitar e dar push para que a alteração seja transimitida;
